openinfo — the HUD

The HUD is a live join: entities detected in what's happening now, matched against everything this context has seen before — and it shows nothing it can't explain.

No mode chips, no queue counters, no engine labels. One dot says it's alive. And the join looks both ways — behind you (what this context has seen) and ahead (what the next context expects of you, checked against evidence). Each item carries its one-line why.

the context index

The display layer is only as smart as the index under it. This is the part that has to be built well once: entity extraction rides the distillation pass it already pays for, and everything above is a ranked join against what accumulates.

Extraction is free-riding

The distill pass already reads every 30 seconds of transcript and every screen diff. The same pass emits structured entities — no second model call. A person mention, a URL on screen, a "I'll send you X" each become a typed record with a pointer back to their moment.

Canon is earned, or pinned

"The SOC 2 link" emerges two ways. Learned: the same file referenced across sessions collapses into one entity, and the version you actually sent to someone outranks versions you merely viewed — outbound use is the strongest canon signal. Pinned: pre-seed a workspace with its canon (compliance folder, runbooks, glossary); pins beat learning, learning fills in whatever you never bothered to pin. And pins are ingested, not bookmarked: a pinned Google Doc or PDF is fetched with read-only auth, chunked with page anchors, and embedded — which is how an answer cites “p. 42” instead of “somewhere in that file.”

Ranking, in one line

score = match × recency × frequency × person-affinity — how strongly the live stream matches the entity, how recently and often it appeared, and whether it was previously exchanged with the people in the room. Dana in the meeting is why Dana's Jun 14 thread outranks a hundred other retention mentions.

People are entities too

A voice or a calendar invite resolves to a person record: sessions shared, things they asked for, things you owe them, things they owe you. That's how "John asked for that report last week" surfaces the moment John speaks — his open asks are indexed against his identity, not against a meeting title.

Display rules the panel lives by

  1. Nothing without a why. Every surfaced item carries the one-line reason it's here. Can't produce the sentence → don't show the card.
  2. Say it once. The context line at the top is the only place the context is named. No mode chips, no engine labels, no status text — one dot is the heartbeat.
  3. Events, not buckets. Moments get individual marks at their own timestamps. Scroll is rewind; the tick rail is the map.
  4. Density follows reality. A dense meeting yields a dense panel; deep work yields three lines. The layout has no minimum to fill.
  5. Plumbing lives elsewhere. Queues, token rates, backends, and mode editing exist — in the settings and analytics surfaces, never on the glass you look through all day.
  6. Blocks, not a layout. The panel is a stack the user owns — collapse it, reorder it, add to it, or build your own block. The defaults are a starter kit, not a verdict.

Three tiers of creativity. Everyone: reorder, collapse, remove, add from the library. Power users: write a query — the same recall() calls the built-ins use, pointed wherever they want. Builders: a custom block is a self-contained HTML surface served by the engine, the rabbithole pattern — the engine thinks, the block renders.
Why this is cheap, not a rewrite. The client never owned data — every built-in block is already an API call against the engine. "Customization" is exposing those same endpoints plus a per-context layout doc (a JSON stack of block configs). The hard part was the seam, and the seam already exists.
Per context, per person. Layouts key to context type — your meeting stack and your deep-work stack differ, and both are yours. Models, density, and palette are block-level settings, so one person's cockpit and another's two-line whisper are the same product.
Scope of this artifact: the HUD as a product — the display layer, the context index that feeds it, the evidence-checked ledger that gives it a future, and the block system that makes it the user's own.
